# SWAG QA Debug Hunter — 新手入門指南

> 你不需要會寫程式，也不需要懂資安。這份指南會帶你從零開始，在 10 分鐘內用上這套工具。

---

## 這個工具能幫我做什麼？

簡單說：**你把程式碼或問題描述貼給 Claude，它會用 SWAG 的業務知識幫你找 Bug、安全漏洞，並給出修復建議。**

它特別懂 SWAG 的四個核心業務：

| 業務 | 它能幫你找的問題舉例 |
|------|------------------|
| 博弈遊戲（龍虎鬥、百家樂） | 賠率算錯、結算重複、亂數不安全 |
| 成人直播（swag.live） | 打賞點數重複扣、訂閱繞過 |
| 金流支付（綠界、微信、支付寶） | 回調沒驗簽、金額被竄改、重複充值 |
| API 買分後台 | 越權調帳、繞過支付直接拿點數 |

---

## 第一步：進入 SWAG QA Debug Hunter Project

1. 打開瀏覽器，進入 [claude.ai](https://claude.ai)
2. 用公司帳號登入
3. 左側選單找到「**Projects**」
4. 點選「**SWAG QA Debug Hunter**」

> 如果找不到這個 Project，請聯繫你的主管或 jason.lee@swag.live 取得加入權限。

---

## 第二步：開始第一個對話

進入 Project 後，點「**New conversation**」開一個新對話。

**你不需要做任何設定**，Project 已經載入好所有 SWAG 業務知識。直接開始問就好。

## 三個最常用的情境

---

### 情境 1：我要審查一段程式碼

工程師開了 PR，你想確認有沒有問題。

**這樣問：**

```
請審查這段程式碼，找出 Bug 和安全問題：

（把程式碼貼在這裡）
```

**你會得到：**

```
## 🔴 BUG-001：支付回調未驗證簽章（高風險）
問題：攻擊者可以偽造回調憑空充值點數
位置：views.py 第 12 行
修復方案：（具體程式碼）

## 🟡 BUG-002：點數更新非原子操作（中風險）
...
```

---

### 情境 2：我要幫新功能寫測試計畫

產品要上線新功能，你需要 QA 測試案例。

**這樣問：**

```
我們下週要上線「用微信支付購買點數」功能。
請幫我產出完整的 QA 測試計畫，包含：
- 正常流程測試
- 異常情境測試
- 安全測試
```

**你會得到：** 完整的測試案例清單，包含步驟、預期結果、邊界條件。

---

### 情境 3：線上出現 Bug，我要分析根因

生產環境報錯，你想快速縮小範圍。

**這樣問：**

```
線上出現問題：用戶打賞 100 點後，點數已從帳戶扣除，
但主播後台顯示沒有收到這筆打賞。
可能的根因有哪些？我應該優先查哪些地方？
```

**你會得到：** 依照 SWAG 直播平台的業務邏輯，列出最可能的原因和排查順序。

## 問問題的小技巧

### ✅ 這樣問效果好

| 技巧 | 範例 |
|------|------|
| 說清楚是哪個業務模組 | 「這是**綠界 ECPay 回調**的程式碼」 |
| 貼上實際的程式碼或錯誤訊息 | 直接貼 code，不要只描述 |
| 說明你想要什麼格式 | 「請產出 Robot Framework 測試腳本」 |
| 一次聚焦一個問題 | 先問安全問題，再問功能 Bug |

### ❌ 這樣問效果差

| 問法 | 為什麼不好 |
|------|----------|
| 「這段 code 有問題嗎？」 | 太模糊，沒說是什麼業務 |
| 「幫我 debug」| 沒提供任何程式碼或錯誤 |
| 「全部幫我測試」| 範圍太大，結果會很籠統 |

## 常見問題 FAQ

**Q：我貼的程式碼會外洩嗎？**
A：請遵守公司資安規定，不要貼含有真實用戶資料、密碼、API Key 的程式碼。測試用的假資料沒問題。

**Q：它給的答案一定是對的嗎？**
A：不一定。Claude 是輔助工具，給的建議要由你判斷是否適用。發現問題的最終確認還是要靠你的專業判斷。

**Q：它可以幫我直接執行測試嗎？**
A：在 claude.ai Project 裡不行，它只能幫你**寫測試腳本**，執行還是要在你的電腦上跑。

**Q：我問了一個問題，答案感覺不對，怎麼辦？**
A：在同一個對話裡繼續問，例如：「你提到的這個修復方法，在我們用 Django 的環境下要怎麼做？」Claude 會根據你的追問調整答案。

## 進階用法（熟悉後再看）

當你用熟了基本問法，可以試試這些指令：

```
# 以攻擊者視角審查
請以攻擊者的角度，找出這段程式碼的所有可利用漏洞

# 產出自動化腳本
請幫我寫一個 pytest 測試，驗證這個 Bug 已經被修復

# 對照規則庫
這段程式碼違反了哪些 SWAG 的 RULE 規則？

# 整合到 PR 流程
這是 PR diff，請給出 QA 審查意見和驗收標準
```
