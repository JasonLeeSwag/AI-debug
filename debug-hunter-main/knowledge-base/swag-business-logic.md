# SWAG 核心業務邏輯（Business Logic）

> QA 必讀：測試前先了解這些業務規則，才能判斷系統行為是 Bug 還是設計如此。
> 來源：Notion 產品文件（Moor Logic、Web Logic）

## 一、Moor 主播端業務邏輯

### 1.1 帳號權限

| 帳號類型 | 說明 |
|---------|------|
| **Mod View** | 超級帳號，無需任何付費解鎖 |
| **一般主播** | 在「一對多」模式中，**無法**自行調整價格 |
| **Non-beta** | 一般直播主帳號權限 |
| **Beta 帳號** | 下了 Beta tag 的帳號；部分 feature 只有 Beta 帳號可預見 |

### 1.2 直播模式定義

| 模式 | 說明 |
|------|------|
| **一般模式** | 免費公開直播 |
| **Show**（Pay Show / 限時表演 / 表演） | 三者視為同等模式 |
| **一對多**（私密直播） | 多位付費觀眾同時觀看 |
| **一對一**（One on One） | 單一用戶與主播私密直播 |

### 1.3 畫面定義
- **History**：結算畫面（一場直播結束後的收益結算頁）
- **收益畫面**：點擊鑽石圖示後的彈出畫面（≠ 結算畫面）

---

### 1.4 邀請機制（Invitations）

| 規則 | 說明 |
|------|------|
| 排他性 | 主播接受任一邀請或自行開 Show → 其他所有邀請自動拒絕 |
| 一對多邀請拒絕 | 主播拒絕 → 所有提出邀請的用戶**一併拒絕** |
| 一對多模式下 | 用戶**不能**再發一對多邀請，但**可以**發一對一邀請 |
| 一對一邀請拒絕 | 主播須**逐一拒絕**（每個用戶分別處理） |
| 一對一模式下 | **不能**再發任何邀請 |

---

### 1.5 一對一（One on One）計費邏輯

> ⚠️ 這是最容易出 Bug 的區域，需嚴格測試計費與退款。

| 時間點 | 行為 |
|-------|------|
| 發起邀請瞬間 | 用戶**立即被預扣**款項 |
| 主播拒絕邀請 | 用戶仍先被預扣（需確認後續退款機制） |
| 主播主動退出 | 用戶可拿回**全部**款項 |
| 正式開始 | 先扣除 **5 分鐘**費用；第 6 分鐘起轉為「每分鐘扣款」 |
| 主播收益結算 | 前 5 分鐘預扣金額在**結算時才加入**主播收益 |

**操作限制**：
- 預扣時間內試圖切換模式 → 跳出阻擋彈窗，**不允許**切換
- 預扣時間內嘗試關播 → 系統跳出提示，但**不阻擋**關播
- 用戶離開提示：超過預扣時間後離開才有提示；預扣時間內離開**無提示**

**Known Issue**：
- 超過預扣時間後關閉直播，History（結算畫面）**不會**加上預扣金額，但後台**實際上有收到**這筆款項。

---

### 1.6 Show（表演模式）機制

| 階段 | 規則 |
|------|------|
| 募集中 | 票價為「早鳥票」 |
| 達標瞬間 | 立刻進入「準備開始 Show」；票價恢復「原價」 |
| 未達標 | 主播仍可隨時選擇開始或結束 Show |
| Show 進行中 | **禁止**：關播、關閉 Show、切換模式、發送挑逗表演 |
| Show 進行中 | 原本公開直播收到的指令**暫停** |
| Show 結束 | 強制轉回「公開模式」 |
| 收益結算 | Show 收益**只在該場結束後**才加入總收益 |
| MVP 判定 | 該場次**買票最多**的用戶 |
| 主播意外退出 | 系統退還全場費用給用戶，但**主播依然可獲得整場費用** |

---

### 1.7 私密直播（一對多）

- 無時間限制，想開多久就開多久
- 支援震動控制與挑逗表演
- 價格：一般主播**無法**自行調整

---

### 1.8 通用功能

**評分系統**：
- 至少累積 **20 次**評分才會顯示在直播畫面
- 一場直播中，單一用戶只能評分 **1 次**
- 分數每分鐘更新一次
- 評分在**所有直播場次中通用**（跨場累計）

**挑逗表演**：
- **不可更改秒數**，只能更改鑽石數量

**封鎖用戶效果**：
| 動作 | 被封鎖用戶 |
|------|----------|
| 送禮 | ❌ 不能 |
| 送指令 | ❌ 不能 |
| 發送邀請 | ❌ 不能 |
| 發文字 | ✅ 可以（但畫面**不顯示**） |
| 觀看免費直播 | ✅ 可以 |

## 二、Web 用戶端業務邏輯

### 2.1 直播間規則

| 功能 | 規則 |
|------|------|
| 1 對 1 | **只能由用戶發起**（主播不能主動發起） |
| 邀請失敗 | 系統退還用戶預扣的鑽石數 |
| 主播主動退出（未達整段時間） | 系統退還對應分鐘數的鑽石數 |
| Hashtag | 至少 **1 個**，最多 **5 個** |

**Show 票券規則**：
- 票券數達標 → Show 成立
- 票券數未達標 → 系統詢問主播是否取消，**預設取消**
- 未完成會「退鑽石」給用戶

**直播間顯示模式**：
- **劇院模式（Theater Mode）**：點擊進入後的全屏/主要模式
- **簡約模式（Compact Mode）**：點擊 `Close Sidebar` 後收起側邊欄

---

### 2.2 影片類型定義

| 類型 | 定義 |
|------|------|
| **長影片（Video Post）** | 3 分鐘以上，60 分鐘以下 |
| **短影音（Shorts）** | 短片 |
| **限時動態（Story）** | 3 秒～30 秒；存在 48 小時；購買後永存於「解鎖項目」 |

---

### 2.3 帳號管理規則

| 欄位 | 規則 |
|------|------|
| 電話號碼 | **可修改**，但**不可刪除** |
| 電子信箱 | **可修改**，但**不可刪除** |
| 鑽石變動日誌 | 任何鑽石增減**必須有記錄可查** |
| 體驗券紀錄 | 需有對應 Log |

---

### 2.4 解鎖項目（Archive / Unlock Items）

- **觸發條件**：只要有「花費」行為，即顯示於解鎖項目
- **顯示邏輯**：顯示主播頭貼 → 點擊進入該主播專屬解鎖內容清單
- **限時動態**：一旦被解鎖後，**永久保存**（超過 48 小時也不消失）

---

### 2.5 VIP 等級

- **舊版**：Lv.0 - Lv.8
- **新版**：Lv.0 - Lv.100（見 GitHub issue #9889）
- **重置週期**：每 **3 個月**重置一次

---

### 2.6 封鎖機制（Block）

- 若主播封鎖用戶：該用戶對該主播**所有花鑽動作均被禁止**
- 主播**無法看到**被封鎖用戶的發言
